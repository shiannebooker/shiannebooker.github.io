What are the differences between presenting an image using an HTML tag (like <img>) and presenting it as the background of an HTML element (using the CSS property background-image)?

The biggest difference that you will see between presenting an image through a tag vs an element is the way they are written and what they are written for. The semantic tag <img> is written within the page’s content, meaning it is another element being added to the page, while presenting an image as the background of an HTML element means that you are changing an element's properties, using CSS.

Besides background-image, what other CSS properties affect the display of background images? Describe at least two more CSS properties.

In CSS, there are a couple of other ways you can alter the display of a background image. You can use background-size to alter the dimensions of the picture. This is super helpful, obviously, so you can display the image however you would like. And you can also use background-repeat to help with positioning the image. By default, the browser will fill the space with the image. You can use background-no-repeat to stop this, or background-repeat-x or background-repeat-y to position the tile.

In one sentence, how would you define the differences between dimensions and resolution in a bitmap image?

Resolution is how tightly packed the pixels are together in a bitmap image, and dimensions is the exact number of pixels in a bitmap image.

What are the similarities and differences between these image file formats? Why are some better than others for specific uses?

  • GIF: A GIF is mostly used for short meme style animations, and its advantage is to be a simple animation. The downfall of this is that GIF is only limited to 256 colors, which can make some GIF's look a little grainy.
  • PNG: A PNG is mostly used for logos or graphics, and it retains every pixel exactly, giving it a super clean look. A weakness of PNG, is that an image is usually going to be a larger file as a PNG, rather than a JPG.
  • JPG, JPEG: A JPEG or JPG is a fast loading, high resolution image that is typically used for photography for social media or websites. A JPEG is better than a PNG because it deletes some of the data that humans don't see or use, so that the image file is a bit smaller.
  • WebP: The WebP was actually created by Google, and their goal was to give users all of the advantages to each file formats. WebP's have the transparency of a PNG, the smaller file size like a JPEG, and the animations like a GIF. The flie sizes are roughly 25%-30% smaller. It's only weakness is that super old browsers may not support it yet.
  • SVG: an SVG is not made of pixels. It is XML code that basically describes points and lines. Which is perfect for what it is most commonly used for. Logos, Icons, and Illustrations. This format is not suitable for photos unfortunately, because it cannot depict the complex colors that go into a photograph.
