How does CSS use color? What do the properties color, background-color, and border-color do?

CSS uses color to ultimately style the elements on a webpage. Color sets the base color of a webpage. It helps a lot with readability and clarity. It sets things like over or underlines to separate specific parts of a text. Background-Color is pretty self-explanatory. It is the background color of the selected elements. The standard background-color is normally transparent, so the designer has the power to do whatever they please. This does not include margins. The border-color property sets the color of the border of the selected element. Unfortunately, this only works if a border style is defined.

How does CSS accept color as a value? Provide examples of colors in these styles: Named, Hexadecimal, RGB, and 1 more that you picked.

CSS accepts color as a value in a few different ways. The most common ways are pre-defined color names, hexadecimal codes, and functional notations. Below are some examples

  • Named: A Named color is basically just a named color. It would look like this <p style=’color:red;’<p>. This would make the defined element red.
  • Hexadecimal: A Hexadecimal code is the ‘hex’ numbers that are commonly used for a lot of things. Examples of these are #ff0000 for red, %00ff00 for green, #0000ff for blue, and #ffffff for white. Alternatively, you can just write #fff for white.
  • RGB: In an RGB code, you kind of get to make the color yourself. You tell the code exactly how much red, blue, and green you want in a color, and it will come out with that color on your website. The numbers range from 0-255, and in RGBA, the A means opacity, so you can also decide that as well.
  • HWB: HWB stands for Hue, Whiteness, and Blackness. Similar to RBG, you kinda make the color yourself. Hue is a number or angle ranging from 0-360 on the colorwheel that decides the initial color. Whiteness and Blackness are both percentages of that color to mix in, so you can make a color as muted or as pale as you want!
